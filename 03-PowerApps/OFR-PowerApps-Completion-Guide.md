Go to **https://make.powerautomate.com**.\n2. Click **+ Create > Scheduled cloud flow**.\n3. Name: `OFR - Daily Staleness Calculator`.\n4. Set **Repeat every: 1 Day**, **Starting: (choose today's date)**, **At: 06:00 AM**, **Time zone: (your time zone)**.\n5. Click **Create**.\n\n### 4.2 Add Actions\n\n#### Step 1 -- Trigger: Recurrence\n\nAlready configured above. Confirm settings:\n\n| Property | Value |\n|----------|-------|\n| Frequency | Day |\n| Interval | 1 |\n| Start time | `2025-01-01T06:00:00Z` (adjust as needed) |\n| Time zone | (your time zone) |\n\n#### Step 2 -- Get Items (OFR_Issues)\n\nAdd action: **SharePoint - Get items**.\n\n| Property | Value |\n|----------|-------|\n| Site Address | `https://[TENANT].sharepoint.com/sites/OFRIssueTracker` |\n| List Name | `OFR_Issues` |\n| Filter Query | `Status ne 'Closed'` |\n| Top Count | `500` |\n\n#### Step 3 -- Apply to Each\n\nAdd action: **Control - Apply to each**.\n\n- **Select an output from previous steps:** `body('Get_items')?['value']`\n\nInside the Apply to Each loop, add the following actions:\n\n#### Step 3a -- Compose: Calculate Days\n\nAdd action: **Data Operations - Compose**.\n\n- **Rename:** `CalcDays`\n- **Inputs:**\n\n```\ndiv(\n    sub(\n        ticks(utcNow()),\n        ticks(items('Apply_to_each')?['LastUpdated'])\n    ),\n    864000000000\n)\n```\n\n> This expression calculates the number of whole days between the LastUpdated date and the current UTC time. `864000000000` is the number of ticks in one day (24 * 60 * 60 * 10,000,000).\n\n#### Step 3b -- Compose: Determine Staleness Flag\n\nAdd action: **Data Operations - Compose**.\n\n- **Rename:** `DetermineFlag`\n- **Inputs:**\n\n```\nif(\n    lessOrEquals(outputs('CalcDays'), 7),\n    'Current',\n    if(\n        lessOrEquals(outputs('CalcDays'), 14),\n        'Aging',\n        'Stale'\n    )\n)\n```\n\n#### Step 3c -- Update Item (OFR_Issues)\n\nAdd action: **SharePoint - Update item**.\n\n| Property | Value |\n|----------|-------|\n| Site Address | `https://[TENANT].sharepoint.com/sites/OFRIssueTracker` |\n| List Name | `OFR_Issues` |\n| Id | `items('Apply_to_each')?['ID']` |\n| DaysSinceUpdate | `outputs('CalcDays')` |\n| StalenessFlag Value | `outputs('DetermineFlag')` |\n\n### 4.3 Complete Flow Summary\n\n```\nRecurrence (Daily 6:00 AM)\n  |\n  v\nGet items (OFR_Issues where Status ne 'Closed')\n  |\n  v\nApply to each (item in Get_items value)\n  |\n  +--> Compose: CalcDays = div(sub(ticks(utcNow()), ticks(item.LastUpdated)), 864000000000)\n  |\n  +--> Compose: DetermineFlag = if(CalcDays <= 7, 'Current', if(CalcDays <= 14, 'Aging', 'Stale'))\n  |\n  +--> Update item: set DaysSinceUpdate = CalcDays, StalenessFlag = DetermineFlag\n```\n\n### 4.4 Save and Test\n\n1. If no items exist, it defaults to 0.\n\n#### Step 5 -- Compose: Next ItemID\n\nAdd action: **Data Operations - Compose**.\n\n- **Rename:** `NextItemID`\n- **Inputs:**\n\n```\nconcat('OFR-', string(add(outputs('MaxItemNumber'), 1)))\n```\n\n#### Step 6 -- Create Item in OFR_Issues\n\nAdd action: **SharePoint - Create item**.\n\n- **Rename:** `Create_Tracker_Item`\n\n| Property | Value |\n|----------|-------|\n| Site Address | `https://[TENANT].sharepoint.com/sites/OFRIssueTracker` |\n| List Name | `OFR_Issues` |\n| Title | `body('Get_Intake_Item')?['Title']` |\n| ItemID | `outputs('NextItemID')` |\n| Owner | `body('Get_Intake_Item')?['Owner']` |\n| Priority Value | `body('Get_Intake_Item')?['Priority']?['Value']` |\n| Status Value | `New` |\n| DateRaised | `utcNow()` |\n| LastUpdated | `utcNow()` |\n| NextAction | `body('Get_Intake_Item')?['Description']` |\n| DaysSinceUpdate | `0` |\n| StalenessFlag Value | `Current` |\n\n#### Step 7 -- Create Initial History Entry\n\nAdd action: **SharePoint - Create item**.\n\n- **Rename:** `Create_History_Entry`\n\n| Property | Value |\n|----------|-------|\n| Site Address | `https://[TENANT].sharepoint.com/sites/OFRIssueTracker` |\n| List Name | `OFR_UpdateHistory` |\n| Title | `outputs('NextItemID')` |\n| ParentItemID | `outputs('NextItemID')` |\n| UpdateDate | `utcNow()` |\n| StatusAtUpdate Value | `New` |\n| Notes | `concat('Issue promoted from intake queue. Original description: ', body('Get_Intake_Item')?['Description'])` |\n| UpdatedBy | `body('Get_Intake_Item')?['Owner']` |\n\n#### Step 8 -- Update Intake Item Status\n\nAdd action: **SharePoint - Update item**.\n\n- **Rename:** `Update_Intake_Status`\n\n| Property | Value |\n|----------|-------|\n| Site Address | `https://[TENANT].sharepoint.com/sites/OFRIssueTracker` |\n| List Name | `OFR_IntakeQueue` |\n| Id | `triggerBody()['text']` |\n| TriageStatus Value | `Promoted` |\n\n#### Step 9 -- Respond to PowerApp\n\nAdd action: **PowerApps (V2) - Respond to a PowerApp or flow**.\n\nClick **+ Add an output > Text**.\n\n| Property | Value |\n|----------|-------|\n| Name | `newitemid` |\n| Value | `outputs('NextItemID')` |\n\n### 5.3 Complete Flow Summary\n\n```\nPowerApps (V2) Trigger\n  Input: IntakeItemID (text)\n  |\n  v\nGet item (OFR_IntakeQueue by IntakeItemID)\n  |\n  v\nGet items (OFR_Issues, order by ItemID desc)\n  |\n  v\nCompose: MaxItemNumber = extract number from top ItemID\n  |\n  v\nCompose: NextItemID = concat('OFR-', MaxItemNumber + 1)\n  |\n  v\nCreate item (OFR_Issues with mapped fields from intake)\n  |\n  v\nCreate item (OFR_UpdateHistory - initial entry)\n  |\n  v\nUpdate item (OFR_IntakeQueue - set TriageStatus to 'Promoted')\n  |\n  v\nRespond to PowerApp (output: newitemid)\n```\n\n### 5.4 Connect Flow to Power Apps\n\n1. Appendix C: Delegation Considerations\n\nPower Apps has a delegation limit (default 500 rows, max 2000) when querying SharePoint. The following formulas in this app are **delegable** to SharePoint:\n\n- `Filter(OFR_Issues, Status.Value <> \"Closed\")` -- delegable\n- `Filter(OFR_IntakeQueue, TriageStatus.Value = \"Pending\")` -- delegable\n- `Filter(OFR_UpdateHistory, ParentItemID = varSelectedIssue.ItemID)` -- delegable\n\nThe following operations are **not delegable** and will only run against the first N rows (where N is your data row limit):\n\n- `varSearch in Title` -- the `in` operator is not delegable for SharePoint\n- `CountRows()` after a non-delegable filter will only count local rows\n\n**Mitigations if you exceed 500 issues:**\n\n1. Increase the data row limit: **File > Settings > General > Data row limit for non-delegable queries** -- set to 2000.\n2. Replace `in` search with `StartsWith()` which is delegable:\n   ```\n   Or(\n       IsBlank(varSearch),\n       StartsWith(Title, varSearch),\n       StartsWith(Owner, varSearch),\n       StartsWith(ItemID, varSearch)\n   )\n   ```\n3. For KPI counts, use a collection populated on screen load:\n   ```\n   // In DashboardScreen.OnVisible:\n   ClearCollect(colAllIssues, OFR_Issues);\n   // Then use: CountRows(Filter(colAllIssues, Status.Value <> \"Closed\"))\n   ```\n\n---\n\n*End of OFR Issue Tracker Power Apps Completion Guide*
